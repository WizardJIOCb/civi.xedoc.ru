# Архитектура Chronicle

## Границы ответственности

`@civi/engine` — чистый детерминированный доменный слой. Он создаёт мир из seed, рассчитывает экономику, календарь, последствия событий и применяет только прошедшие валидацию действия. Пакет не знает о HTTP, PostgreSQL или OpenRouter.

`@civi/server` — оркестратор. Он выбирает державу для очередного решения, передаёт модели компактный контекст, валидирует JSON, вызывает движок, сохраняет replay-снимки и транслирует мир через WebSocket.

`@civi/web` — observer UI. Клиент получает неизменяемый `WorldSnapshot`; управляющие команды меняют только режим исполнения симуляции, но не игровые параметры.

## Поток одного хода

1. Сервер берёт snapshot N и выбирает державу по round-robin.
2. Decision provider получает цивилизацию, соседей, отношения, память и активные события.
3. OpenRouter возвращает JSON по строгой схеме. При недоступности провайдера включается воспроизводимый fallback за пределами движка.
4. Engine проверяет action type, investment и все target ID.
5. Правила мира рассчитывают последствия и создают snapshot N+1 с checksum.
6. Решение сохраняется, каждый четвёртый snapshot записывается в PostgreSQL, клиенты получают WebSocket-сообщение.

## Детерминизм и replay

Псевдослучайность выводится только из `seed + subsystem + tick + entityId`. Внешнее AI-решение логируется целиком и повторно подаётся в engine при replay. SHA-256 checksum покрывает состояние цивилизаций, отношений и событий.

## Каталог событий

32 архетипа умножаются на 16 географических контекстов — 512 сценариев с уникальными ID. Архетип определяет факты, scope, tone, severity и допустимые намерения, но не реакцию державы. Расширение каталога не требует менять simulation loop.
